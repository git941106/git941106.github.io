简介

web自动化，通常是通过调用浏览器驱动，对浏览器进行模拟用户行为的操作，采用python+selenium+allure技术，实现自动化

安装

windows

  1. python
  2. pytest 安装，pip install pytest
  3. selenium安装，pip install selenium
  4. 下载对应的浏览器驱动，驱动最好配置环境变量

linux

  1. 安装浏览器
  2. 安装浏览器对应驱动
  3. python，linux环境默认安装python2
  4. pip，默认没有安装pip

注意

如果使用selenium自定义的函数无法解决的问题，可考虑使用js解决

selenium基本用法

js中window.scrollTo(x,y),x表示横向移动，y表示纵向移动，这个操作的是整个页面，selenium的详情，可访问官网

滑动滚动条

窗口的滚动条与元素的滚动条滑动方法不一样，元素的滚动条可以先定位元素，窗口的滚动条通过window的属性滑动，元素滚动条的特殊情况下，滚动条是一个div元素，则直接定位此元素

滚动到底部

driver.execute_script("window.scrollTo(0,document.body.scrollHeight)")

滚动到顶部

driver.execute_script("window.scrollTo(0,0)")

移动到指定元素

driver.execute_script("arguments[0].scrollIntoView(false);", ele)

    print(driver.window_handles)
    # 记录当前窗口
    handle = driver.current_window_handle
    # 切换到新开的窗口
    driver.switch_to.window(driver.window_handles[-1])
    print(driver.title)
    # 关闭窗口
    driver.close()
    # 切换回新开的窗口
    driver.switch_to.window(handle)
    print(driver.title)
    sleep(1)
    driver.quit()

等待

等待分类：

隐式等待，初始化浏览器一般会设置隐式等待。

显示等待：有时候隐式等待不生效或者其他因素，可使用显示等待。一般在后台的管理系统中，添加操作的时候，会有一个 添加成功或者失败的提示，这个提示是会停留2s左右自动消失的，如果在自动化的过程中不添加等待的话，会获取到之前的未消失的结果。

强制等待：不推荐使用，如果可以，推荐使用显示等待。

from time import sleep
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support import expected_conditions
from selenium.webdriver.support.wait import WebDriverWait
if __name__ == '__main__':
    driver = webdriver.Chrome()
    driver.maximize_window()
    driver.get("https://music.163.com/")
    print(driver.title)
    # 显示等待，在3s内，轮询判断title成立，返回True,否则报超时异常，常用
    print(WebDriverWait(driver, 3).until(expected_conditions.title_is("网易云音乐")))
    # 显示等待，在3s内，轮询判断title不成立，返回False,否则报超时异常
    print(WebDriverWait(driver, 3).until_not(expected_conditions.title_contains("网易音乐")))
    sleep(1)# 强制等待
    driver.quit()

def get_cookies(cookies:str)->list:
    '''
    将字符串的类型的cookies，转换成列表，列表元素是字典{name:"",value:""}形式
    '''
    # 切割成列表
    cookies = cookies.split(";")
    # 去除前后空格
    for i in range(len(cookies)):
        cookies[i] = cookies[i].strip()
    # 修改列表元素，driver.add_cookie(i)方法的参数是字典，key必须包含name和value
    # BD_UPN=12314753元素，name就是BD_UPN,value就是12314753
    for i in range(len(cookies)):
        cookies[i] = {'name': cookies[i].split("=")[0], 'value': cookies[i].split("=")[1]}
    return cookies
@pytest.fixture(scope="class",autouse=True)
def login():
    cookies = ''#填写复制的cookies字符串
    cookies = get_cookies(cookies)

    driver = BrowserUtil.get_driver()
    # 必须先打开网页，才能添加cookies
    driver.get("https://www.baidu.com")
    for i in cookies:
        driver.add_cookie(i)
    # 添加cookies后，需要刷新网页
    driver.refresh()

browser_util.py

from selenium import webdriver
from selenium.webdriver.remote.webdriver import WebDriver


class BrowserUtil:
    driver: WebDriver = None

    @classmethod
    def get_driver(cls):
        if cls.driver is None:
            # 初始化浏览器驱动
            cls.driver = webdriver.Chrome()
            # 最大化浏览器
            cls.driver.maximize_window()
            # 设置隐式等待10s
            cls.driver.implicitly_wait(10)
        return cls.driver

    @classmethod
    def quit_browser(cls):
        if cls.driver is not None:
            cls.driver.quit()
            cls.driver = None

复用浏览器

复用浏览器，必须先关闭浏览器（否则执行脚本会报错或出现其他问题），然后使用命令打开一个新的浏览器，使用命令行打开浏览器，需要设置浏览器的路径到环境变量中，使用命令行打开的浏览器无法使用driver.quit()退出浏览器，不过可以使用driver.close()关闭打开的所有页面，浏览器就自动退出了

chrome --remote-debugging-port=8888 
# 命令行打开浏览器,端口没有特殊要求，不适用系统已占用端口和已定义端口即可
# 验证服务是否成功，可在打开的浏览器中输入ip:port,一般是127.0.0.1:8888(127.0.0.1是本机ip，8888是之前
# 设置的端口号)

特殊元素定位技巧

当添加元素成功，弹出的消息，2s左右就消息，或者需要悬停鼠标才会出现，鼠标移开又消失的元素，这些元素的定位可以借助js的定时函数辅助定位，f12打开开发者工具，console中输入setTimeout(“debugger”,5000)，按下enter，5s就会执行debugger，页面会冻结，进入debugger模式，这时就可以进行定位

常见问题

元素找不到

  1. 查找的条件是否写对，直接在浏览器器中f12，ctrl+f，在查找框输入条件，确认是否可查到元素
  2. 查找的页面是否对，有时候代码执行太快，需要加等待，或者页面刷新，这是会报元素没有附到当前页面，需要注意加等待或者加判断条件，判断其他的元素出现才进行查找
  3. 判断是否有frame框架，有frame需要切到frame中
  4. 判断元素的属性，如id，class是否是动态的，需要切换其他的定位方式，如通过tag，name等定位

元素不可交互

元素不可交互说明元素已经定位到，但是由于是隐藏的元素，不能click，send_keys等元素操作。可以按正常的操作步骤，让元素显示之后再定位操作元素，或者通过js，移除元素的隐藏属性，操作元素
