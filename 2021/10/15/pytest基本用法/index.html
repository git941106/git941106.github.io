简介

pytest是python的第三方模块，用于自动化测试

接口自动化测试：pytest+requests+allure

webui自动化测试：pytest+selenium+allure

app自动化测试：pytest+appium+allure

基本用法

前提条件

  • 已安装python
  • 已安装pip(python的模块，安装python时自动安装)

安装pytest

  • pip install pytest

查看是否安装成功

  • windows：pip list | findstr pytest
  • mac : pip list | grep pytest

测试用例的识别与运行

测试文件
  • test_*.py或者*_test.py
用例识别
  • Test*类包含的所有test_*方法(测试类不能带有_init_方法)
  • 不在class中的所有的test_*方法，不能识别*_test方法
  • pytest也可以执行unittest框架写的用例和方法
  • 所有的包必须有__init__.py

常用命令

查看帮助文档

pytest -h
pytest --help
执行用例
pytest 执行当前目录及其子目录下所有符合要求的用例
pytest *.py 执行指定某个测试文件
pytest 文件名/ 执行某个目录下的测试文件
pytest -m smoke 运行@pytest.mark.smoke 标志的用例
遇到错误时停止
pytest -x 遇到一个错误就停止运行
pytest --maxfail=2 遇到错误达到指定数2时停止运行

注意调用顺序

setup_module>setup_class>setup_method>setup>teardown>teardown_method>teardown_class>teardown_module

一般setup_method与setup选择其中一个即可，都是方法调用之前调用

pytest.ini

[pytest]
log_cli = true
log_level = info
log_date_format = %Y-%m-%d %H:%M:%S
log_format = %(asctime)s (%(filename)s:%(lineno)s) %(levelname)-8s %(message)s
addopts=-vs --alluredir=./report --clean-alluredir

pytest -h 可查看相关key

log_cli 表示控制台实时打印日志,默认false

log_cli_level与log_level的区别，cli指的是客户端的控制台，没有cli的优先级比较大，log_level，配合allure，表示allure的日志输出等级,日志的级别有debug>info>warning>error，设置最大时，所有日志都会打印，默认warning

addopts表示执行执行pytest 后的参数，-s 控制台显示print打印的信息，allure不会收集stout信息, –alluredir=./report allure报告相关数据的生成位置，–clean-alluredir 清空之前的数据

Template替换yaml中变量

Template基础

$变量名 或者${变量名}（推荐使用）都可以替换safe_substitute方法只替换传入的变量，没有传入的保持默认，如下面的$what，另一个方法substitute，必须传入对应的参数，否则会报错

from string import Template
template = Template("$what is your name? i am ${name}") # 创建Template对象
name = {"name":"meng"}
print(template.safe_substitute(name))

hook函数

项目根路径的conftest.py文件中

用例名称与路径支持中文
# from _pytest import hookspec 存在_pytest下的hookspec文件中，pycharme在site-package下
def pytest_collection_modifyitems(session, config, items):
    for item in items:
        # item.name 用例的名字
        item.name = item.name.encode('utf-8').decode('unicode-escape')
        # item.nodeid 用例的路径
        item._nodeid = item.nodeid.encode('utf-8').decode('unicode-escape')
用例执行失败自动截图
# 从runner点进去,搜索pytest_runtest_makereport,连续点击后进入listener文件中，复制函数，此函数会被执行# 三次，分别是setup，call，teardown，call就是要执行的用例，截图需要driver，就是需要一个工具类获driver
@pytest.hookimpl(hookwrapper=True)
def pytest_runtest_makereport(item, call):
    '''
    获取每个用例状态的钩子函数
    :param item:
    :param call:
    :return:
    '''
    # 获取钩子方法的调用结果,这是原函数中的语句
    report = (yield).get_result()
    # 仅仅获取用例call 执行结果是失败的情况, 不包含 setup/teardown
    if report.when == "call" and report.failed:

        with allure.step('添加失败截图...'):

            allure.attach(DriverUtil.get_driver().get_screenshot_as_png(), "失败截图", allure.attachment_type.PNG)
